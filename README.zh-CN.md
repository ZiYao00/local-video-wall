# Local Video Wall

[English README](README.md)

一个可以同时预览多个本地视频的瀑布流视频墙工具。

![预览](assets/preview01.gif)

Local Video Wall 可以把本地文件夹变成一个浏览器里的视频墙。它最核心的能力是：在一个页面里同时显示多个本地视频，并让当前屏幕内的视频同时静音播放，方便你不用一个个打开文件，也能快速筛选大量视频。

它也支持图片、批量整理、回收站移动和 AI 生成元数据查看，所以也可以作为本地 AI 素材筛选工具使用。它不是云相册，不是通用文件管理器，也不是 ComfyUI 插件。

## 为什么需要这个工具？

如果你经常浏览本地视频或 AI 图片 / 视频，很快会遇到这些问题：

- 一个文件夹里有几百到几千个短视频和图片，资源管理器很难快速预览。
- 视频需要一个个点开，筛选效率很低。
- 你想同时看多个视频的动态效果，而不是一个播放器里只看一个文件。
- 好图、废图、参考素材和待整理素材混在一起，后期很难回看。
- Prompt、模型、LoRA、workflow 信息不方便在文件夹里直接查看。
- 不想把本地素材上传到云相册或在线素材库。
- 批量筛选时直接删除有风险，希望整理操作尽量可撤回。

## 这个工具怎么解决？

工具会把本地文件夹变成一个瀑布流视频 / 媒体墙：

- 自动扫描本地视频和图片，并以网格方式展示。
- 当前屏幕内的多个视频可以同时自动静音播放。
- 离屏媒体会暂停或释放资源，减少浏览器和硬盘压力。
- 支持搜索、排序、筛选、收藏、批量选择和 CSV 导出。
- 支持图片大图预览和全屏幻灯片。
- 支持视频大窗口播放、上一段/下一段、滚轮音量、全屏和播放模式。
- 支持查看常见 AI 生成元数据，包括正面提示词、负面提示词、模型、LoRA 和 ComfyUI workflow。
- 文件整理保持本地、可见、可撤回，移到回收站时使用 Windows 系统回收站。

## 核心能力

- **多视频瀑布流预览**：在一个页面里同时浏览多个本地视频，并让可见视频同时自动播放。
- **本地 AI 素材墙**：浏览本地下载视频素材、AI 生成图片 / 视频、ComfyUI 输出、A1111 图片、Wan / Kling / Runway 视频和类似本地文件夹。
- **大量文件快速筛选**：支持 2-20 列、每页最高 240 个卡片、顶部翻页、可选悬浮分页条，以及 500MB 以上视频占位。
- **面向性能的可见区域加载**：可见视频按设置上限自动播放，离屏媒体暂停或释放资源，图片接近可见区域才加载，不默认生成磁盘缩略图缓存。
- **整理与审核流程**：收藏、工具栏筛选、批量选择、CSV 导出、文件名排除规则，以及安全移动到 Windows 回收站。
- **图片和视频预览**：图片大图预览、全屏幻灯片、淡入淡出 / 滑动 / 动态漂移效果；视频预览支持上一段/下一段、循环 / 顺序 / 随机、全屏和默认应用打开。
- **AI 生成元数据查看**：读取 A1111 PNG `parameters`、ComfyUI PNG `prompt` / `workflow`、同名 sidecar JSON、LoRA 强度、UNET / diffusion 模型加载器名称，以及可选 `ffprobe` 视频 tags。
- **Windows 本地优先**：支持 `start.bat` 手动启动、`service.bat` 服务菜单、本地 `config.json`、本地 review 数据、默认英文界面和中文切换。

## 使用场景

这个工具适合：

- 浏览包含大量本地短视频的文件夹
- 不想一个个打开播放器，而是想同时预览多个视频
- 浏览 AI 生成的视频和图片批次
- 浏览 ComfyUI / Stable Diffusion / Wan / Kling / Runway 等工具生成的本地输出
- 快速筛选大量本地媒体文件，不需要导入云端服务
- 像看动态素材墙一样浏览本地文件夹
- 查看生成图片的提示词、模型、LoRA 和 workflow 信息
- 为后续标签、元数据搜索、提示词和模型信息整理做准备

## 适合 / 不适合

适合：

- 本地有很多视频，想一次预览多个视频。
- 想快速筛选生成视频、下载视频素材或图片 / 视频混合文件夹。
- 需要收藏、批量选择、CSV 导出、移到回收站等简单本地整理操作。
- 希望在有元数据时查看 AI 生成提示词、模型、LoRA 和 workflow。

不适合：

- 想在线播放网络视频。
- 想做专业视频剪辑。
- 想做云相册、多人素材库或远程同步系统。
- 本地没有需要浏览的视频或图片文件。

## 运行环境

- Windows 10 / Windows 11
- Python 3.10 或更高版本，推荐 Python 3.12
- 推荐使用 Chrome / Edge 浏览器

这个工具会以本地 Python 服务运行。如果电脑没有安装 Python，`start.bat` 和 `service.bat` 不能直接启动工具。

### 新手 Python 检查

1. 从 Python 官网或 Microsoft Store 安装 Python。
2. 如果安装器显示 `Add python.exe to PATH`，建议勾选。
3. 打开命令提示符或 PowerShell，运行：

```bash
python --version
```

或：

```bash
py --version
```

如果这两个命令都无法显示 Python 版本，说明 Windows 找不到 Python，`.bat` 启动文件也无法正常启动工具。

## 使用方法

### 1. 下载

普通用户建议：

1. 打开 GitHub Releases 页面。
2. 下载最新的 `local-video-wall-*.zip` 压缩包。
3. 解压到普通本地文件夹，例如：

```text
C:\Tools\local-video-wall
```

4. 运行 `start.bat`。

开发者也可以克隆仓库，并从源码运行。

### 2. 启动工具

手动模式：

双击：

```text
start.bat
```

浏览器会自动打开，并显示命令行窗口。关闭这个命令行窗口就等于退出工具。

服务模式：

```text
service.bat
```

这个文件会打开英文菜单，可以选择后台启动、停止或重启后台服务、安装开机自启、取消开机自启、打开浏览器、后台启动并打开浏览器或检查状态。开机启动会使用 Windows APPDATA 目录下的稳定辅助脚本，不再依赖临时目录。

也可以在命令行运行：

```bash
python app.py
```

然后访问：

```text
http://127.0.0.1:8787
```

### 3. 如果无法启动

优先检查这些问题：

- 如果 Windows 提示找不到 `python`，需要安装 Python，并确认 Python 已加入 PATH。
- 如果浏览器没有自动打开，可以手动访问 `http://127.0.0.1:8787`。
- 如果网页打不开，可以运行 `service.bat`，选择状态检查服务是否正在运行。
- 如果 `8787` 端口被其它程序占用，可以关闭占用程序，或修改 `app.py` 里的 `PORT` 后再启动。
- 如果 Windows SmartScreen、杀毒软件或浏览器弹出安全提示，请确认压缩包来源可信，再允许本地脚本运行。
- 如果墙内视频没有声音，这是浏览器自动播放规则导致的正常现象；为了允许自动播放，墙内视频默认静音。
- 视频元数据读取需要系统 PATH 里能找到 `ffprobe`。没有 `ffprobe` 时工具仍然能正常运行，只是部分视频元数据字段不可用。

### 4. 选择媒体目录

你可以：

- 手动输入视频文件夹路径，例如 `C:\Users\你的用户名\Videos`
- 点击侧边栏按钮，打开左侧分栏文件夹树
- 使用路径栏内部的星标按钮收藏当前路径
- 使用路径栏内部的历史按钮打开最近路径或清空历史记录
- 点击“Choose Folder / 选择文件夹”按钮
- 勾选“Remember path / 记住路径”
- 勾选“Scan subfolders / 扫描子文件夹”

从文件夹面板或 Windows 文件夹选择框选中路径后，会自动填入路径栏并开始扫描。手动输入路径时，可以按 Enter 或点击“Scan / 扫描”。

### 5. 浏览和整理图片 / 视频

- 使用顶部工具栏切换“全部 / 视频 / 图片 / 收藏”。
- 使用列数选择器调整媒体墙密度。2-9 列时，可见视频会按设置里的“墙内播放上限”自动播放；10 列及以上使用静态墙内预览。
- 使用每页数量输入框控制每页渲染的卡片数量，最高 240。
- 使用顶部翻页箭头、可选悬浮分页条，或在关闭悬浮分页条时使用底部分页。
- 点击图片可以打开图片预览窗口。
- 图片预览里可以使用左右按钮、键盘方向键或鼠标滚轮切换上一张/下一张。
- 点击“幻灯片（全屏）”可以进入图片幻灯片模式。
- 幻灯片模式下，可以在顶部工具栏控制播放/暂停、间隔、效果、Fit、循环和完整屏幕。
- 点击视频可以打开视频预览窗口。
- 视频预览里可以使用左右按钮切换上一段/下一段。
- 视频预览里可以使用鼠标滚轮调整音量。
- 视频播放模式支持单视频循环、顺序播放和随机播放。
- 图片、幻灯片和视频的观看控制栏会自动隐藏；需要操作时，把鼠标移到右上角控制区域即可重新显示。
- 点击批量按钮后，可以选择多个可见媒体，并执行收藏、取消收藏、移到回收站或导出 CSV。
- 在预览右侧的元数据面板里，可以查看文件信息、正面提示词、负面提示词、大模型、LoRA 列表及强度、来源/路径，以及原始元数据操作。
- 对 ComfyUI 图片，工具会优先从 KSampler 的 conditioning 链路提取主提示词，尽量避免 FaceDetailer、未连接或未使用文本节点污染正面提示词。
- 对 ComfyUI workflow，可以从元数据操作里复制 workflow JSON，或打开本地 ComfyUI 页面。

### 6. 界面选项

顶部工具栏和设置菜单提供这些常用控制：

- 语言：英文 / 中文
- 主题：暗色 / 亮色
- 布局位置：居中 / 左侧 / 右侧
- 路径历史：可清空最近路径，收藏路径会保留
- 收藏路径和历史记录都支持单条移除
- 文件名排除规则：最多添加 30 个关键词，不区分大小写，可随时启用、停用或删除
- 大视频墙内预览：500MB 以上默认关闭，但点击占位封面仍可按需打开播放
- 墙内播放上限：设置当前可见视频中最多同时自动播放多少个
- 悬浮分页条：滚动时显示临时分页控制；开启后底部分页会自动隐藏
- 界面字体：小 / 标准 / 大
- 网格列数：2-20 列
- 每页数量：1-240 个卡片
- 顶部筛选：全部 / 视频 / 图片 / 收藏

## 推荐设置

普通浏览和预览：

```text
6列
每页 120
```

看细节：

```text
4列 或 5列
每页 80-120
```

快速扫素材：

```text
8列 或 9列：适合自动播放预览
10-20列：适合静态总览
```

## 注意事项

浏览器自动播放视频通常要求视频静音，所以瀑布流里的视频默认是静音播放。500MB 以上的视频默认在墙内显示占位封面，避免误加载电影或演唱会等超大文件；可在设置中修改。

点击视频后，会打开大窗口播放器，你可以在播放器里手动开启声音。

视频文件的内嵌生成元数据取决于视频生成工具是否真的把信息写进容器。如果系统 PATH 中可以找到 `ffprobe`，工具会尝试读取 MP4/WebM/MOV 容器 tags，例如 prompt、workflow、comment、description、时长、编码和分辨率。如果没有安装 `ffprobe`，仍会安全回退到同名 sidecar JSON 和基础文件信息。

## 本地与安全说明

- 工具以本地 Python 服务运行，地址是 `http://127.0.0.1:8787`。
- 工具不会把你的媒体文件上传到云端。
- 设置保存在 `config.json`，其中可能包含你的本地文件夹路径。
- 审核数据保存在本地 `review_data.json`。
- 移到回收站会使用 Windows 系统回收站；只要回收站没有被禁用或清空，通常可以从 Windows 回收站恢复。
- 500MB 以上视频默认不会在墙内直接预览，避免误加载大型电影或演唱会文件。
- 默认不会生成持久化图片缩略图缓存或视频封面缓存。

## 配置文件

工具会在本地使用：

```text
config.json
```

保存你的设置，例如路径、列数、每页数量、墙内播放上限、悬浮分页条、排序方式、沉浸模式、语言、主题和字体大小。

幻灯片的间隔、效果、Fit 模式和循环设置也会保存在 `config.json`。

这个文件可能包含你的本地路径，不建议上传到 GitHub。上传仓库时保留 `config.example.json` 即可。

## 未来方向

项目已经从基础媒体墙推进到 `v1.8.x` AI 元数据预览阶段。当前核心 PNG 生成元数据预览已经可用，并且在系统安装 `ffprobe` 时可选读取视频容器元数据。

建议顺序：

1. 完成 `v1.8.x - Metadata Stabilization`：测试更多 ComfyUI 节点格式、优化元数据面板可读性，并验证更多真实视频元数据样本。

3. 开发 `v2.0.0 - Local Index`：增加轻量 SQLite 索引，用于加速重复扫描和元数据查询。
4. 开发 `v2.1.0 - Metadata Search`：按 Prompt、模型、LoRA、标签、来源 URL 和审核状态搜索筛选。
5. 视情况开发 `v2.2.0 - Optional Preview Cache`：在元数据和索引稳定后，再考虑可选图片缩略图和视频封面缓存。
6. 改进 `v2.3.0 - Packaging`：Release ZIP、启动检查、可选 portable tools 和更清晰的故障排查。

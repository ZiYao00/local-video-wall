# Local Video Wall

[English README](README.md)

一个面向 AI 图片 / 视频素材的本地浏览、筛选、整理与元数据查看工具。

![预览](assets/preview01.gif)

Local Video Wall 可以把你的本地文件夹变成浏览器里的媒体墙：不用一个个打开文件，就能同时预览大量视频和图片，并快速完成筛选、收藏、批量整理、回收站移动和生成信息查看。

它不是云相册，不是通用文件管理器，也不是 ComfyUI 插件。它的目标更明确：作为一个本地 AI 图片 / 视频素材的快速 review、筛选、标记和找回工具。

## 为什么需要这个工具？

如果你经常生成、下载或整理 AI 图片 / 视频，很快会遇到这些问题：

- 一个文件夹里有几百到几千个图片和短视频，资源管理器很难快速浏览。
- 视频需要一个个打开，筛选效率很低。
- 你想同时比较多个视频的动态效果，而不是在播放器里逐个切换。
- 好图、废图、参考素材、未完成实验混在一起，后期很难回看。
- Prompt、Negative Prompt、模型、LoRA、workflow 信息不方便在文件夹里直接查看。
- 不想把本地素材上传到云相册或在线素材库。
- 批量整理时直接删除有风险，希望文件操作尽量可撤回。

## 它怎么解决？

Local Video Wall 会把本地文件夹变成一个瀑布流图片 / 视频媒体墙：

- 自动扫描本地视频和图片，并以网格方式展示。
- 当前屏幕内的多个视频可以同时自动静音播放。
- 离屏媒体会暂停或释放资源，减少浏览器和硬盘压力。
- 支持搜索、排序、筛选、收藏、批量选择和 CSV 导出。
- 支持图片大图预览和全屏幻灯片。
- 支持视频大窗口播放、上一段 / 下一段、滚轮音量、全屏和播放模式。
- 支持查看常见 AI 生成元数据，包括 Prompt、Negative Prompt、模型、LoRA 和 ComfyUI workflow。
- 整理废片时先移动到可恢复的项目内回收文件夹，再按需移到 Windows 回收站。

## 典型使用流程

1. 运行 `start.bat` 或 `service.bat` 启动工具。
2. 选择一个本地素材文件夹，例如 ComfyUI 输出目录、Civitai 下载目录或视频生成目录。
3. 设置媒体墙密度，例如普通浏览用 6 列，快速扫素材用 8-9 列。
4. 在媒体墙里混合浏览图片和视频。
5. 点击有价值的素材，进入图片或视频预览。
6. 查看 Prompt、Negative Prompt、模型、LoRA、workflow 和文件信息。
7. 收藏可用素材，批量选择废片，导出 CSV，或把不需要的文件移到可恢复的回收文件夹。

## 核心能力

### 多视频同时预览

在一个页面里同时浏览多个本地视频。当前屏幕内的视频可以自动静音播放，滚出屏幕后自动暂停或释放资源。

### 图片 / 视频混合素材墙

支持浏览本地下载视频素材、AI 生成图片 / 视频、ComfyUI 输出、Stable Diffusion WebUI / A1111 图片、Wan / Kling / Runway 视频和类似本地文件夹。

### 快速筛选与批量整理

支持工具栏筛选、搜索、排序、收藏、批量选择、CSV 导出、文件名排除规则，以及安全移动到可恢复的回收文件夹。

### 可恢复的回收文件夹

文件会先移动到扫描目录内的 `_video_wall_trash`。回收站页面以静态图片 / 视频卡片显示素材，支持全部 / 视频 / 图片筛选和多选，可恢复原文件或移到 Windows 系统回收站。回收站中的视频停留在静帧，不生成持久化预览缓存。

### AI 生成元数据查看

在有元数据时，可以读取并展示：

- Stable Diffusion WebUI / A1111 PNG `parameters`
- ComfyUI PNG `prompt` 和 `workflow`
- 同名 sidecar JSON
- Prompt 和 Negative Prompt
- 模型、UNET / diffusion 模型加载器名称
- LoRA 名称和强度信息
- 媒体卡片上的 workflow / 生成信息角标，可区分检测中、含生成信息、仅有 workflow 的文件
- 可选通过 `ffprobe` 读取视频容器 tags

对 ComfyUI 图片，工具会优先从 KSampler 的 conditioning 链路提取主提示词，尽量减少 FaceDetailer、未连接节点或未使用文本节点对主提示词的污染。

### Windows 本地优先

工具以本地 Python 服务运行。设置和 review 数据保存在本地。支持 `start.bat` 手动启动，也支持通过 `service.bat` 使用后台服务菜单。

## 适合 / 不适合

| 适合 | 不适合 |
| --- | --- |
| 本地有很多短视频，想一次预览多个 | 在线播放网络视频 |
| 快速筛选 AI 图片 / 视频批次 | 专业视频剪辑 |
| 浏览 ComfyUI / A1111 / Wan / Kling / Runway 输出 | 云相册或远程同步服务 |
| 查看 Prompt、模型、LoRA 和 workflow 元数据 | 多人素材管理系统 |
| 本地收藏、批量选择、CSV 导出、回收站整理 | 本地没有需要浏览的媒体文件 |

## 运行环境

- Windows 10 / Windows 11
- Python 3.10 或更高版本，推荐 Python 3.12
- 推荐使用 Chrome / Edge 浏览器

这个工具会以本地 Python 服务运行。如果电脑没有安装 Python，`start.bat` 和 `service.bat` 不能直接启动工具。

### 新手 Python 检查

1. 从 Python 官网或 Microsoft Store 安装 Python。
2. 如果安装器显示 `Add python.exe to PATH`，建议勾选。
3. 打开命令提示符或 PowerShell，运行：

```bash
python --version
```

或：

```bash
py --version
```

如果这两个命令都无法显示 Python 版本，说明 Windows 找不到 Python，`.bat` 启动文件也无法正常启动工具。

## 使用方法

### 1. 下载

普通用户建议：

1. 打开 GitHub Releases 页面。
2. 下载最新的 `local-video-wall-*.zip` 压缩包。
3. 解压到普通本地文件夹，例如：

```text
C:\Tools\local-video-wall
```

4. 运行 `start.bat`。

开发者也可以克隆仓库，并从源码运行。

### 2. 启动工具

手动模式：

```text
start.bat
```

浏览器会自动打开，并显示命令行窗口。关闭这个命令行窗口就等于退出工具。

服务模式：

```text
service.bat
```

这个文件会打开服务菜单，可以选择后台启动、停止或重启后台服务、安装开机自启、取消开机自启、打开浏览器、后台启动并打开浏览器或检查状态。

也可以在命令行运行：

```bash
python app.py
```

然后访问：

```text
http://127.0.0.1:8787
```

### 3. 如果无法启动

优先检查这些问题：

- 如果 Windows 提示找不到 `python`，需要安装 Python，并确认 Python 已加入 PATH。
- 如果浏览器没有自动打开，可以手动访问 `http://127.0.0.1:8787`。
- 如果网页打不开，可以运行 `service.bat`，选择状态检查服务是否正在运行。
- 如果 `8787` 端口被其它程序占用，可以关闭占用程序，或修改 `app.py` 里的 `PORT` 后再启动。
- 如果 Windows SmartScreen、杀毒软件或浏览器弹出安全提示，请确认压缩包来源可信，再允许本地脚本运行。
- 如果墙内视频没有声音，这是浏览器自动播放规则导致的正常现象；为了允许自动播放，墙内视频默认静音。
- 视频元数据读取需要系统 PATH 里能找到 `ffprobe`。没有 `ffprobe` 时工具仍然能正常运行，只是部分视频元数据字段不可用。

### 4. 选择媒体目录

你可以：

- 手动输入视频文件夹路径，例如 `C:\Users\你的用户名\Videos`。
- 点击侧边栏按钮，打开左侧分栏文件夹树。
- 使用路径栏内部的星标按钮收藏当前路径。
- 使用路径栏内部的历史按钮打开最近路径或清空历史记录。
- 点击“Choose Folder / 选择文件夹”按钮。
- 勾选“Remember path / 记住路径”。
- 勾选“Scan subfolders / 扫描子文件夹”。

从文件夹面板或 Windows 文件夹选择框选中路径后，会自动填入路径栏并开始扫描。手动输入路径时，可以按 Enter 或点击“Scan / 扫描”。

### 5. 浏览和整理图片 / 视频

- 使用顶部工具栏切换“全部 / 视频 / 图片 / 收藏”。
- 使用列数选择器调整媒体墙密度。
- 2-9 列时，可见视频会按设置里的“墙内播放上限”自动播放。
- 10 列及以上使用静态墙内预览。
- 使用每页数量输入框控制每页渲染的卡片数量，最高 240。
- 使用顶部翻页箭头、可选悬浮分页条，或在关闭悬浮分页条时使用底部分页。
- 点击图片可以打开图片预览窗口。
- 图片预览里可以使用左右按钮、键盘方向键或鼠标滚轮切换上一张 / 下一张。
- 点击“幻灯片（全屏）”可以进入图片幻灯片模式。
- 点击视频可以打开视频预览窗口。
- 视频预览里可以使用左右按钮切换上一段 / 下一段。
- 视频预览里可以使用鼠标滚轮调整音量。
- 视频播放模式支持单视频循环、顺序播放和随机播放。
- 点击批量按钮后，可以选择多个可见媒体，并执行收藏、取消收藏、移到回收站或导出 CSV。
- 在预览右侧的元数据面板里，可以查看文件信息、Prompt、Negative Prompt、大模型、LoRA 列表及强度、来源 / 路径，以及原始元数据操作。
- 对 ComfyUI 图片，工具会优先从 KSampler 的 conditioning 链路提取主提示词，尽量避免 FaceDetailer、未连接或未使用文本节点污染正面提示词。
- 对 ComfyUI workflow，可以从元数据操作里复制 workflow JSON，或打开本地 ComfyUI 页面。

### 6. 界面选项

顶部工具栏和设置菜单提供这些常用控制：

- 语言：英文 / 中文
- 主题：暗色 / 亮色
- 布局位置：居中 / 左侧 / 右侧
- 路径历史和收藏路径
- 文件名排除规则
- 大视频墙内预览
- 墙内播放上限
- 悬浮分页条
- 界面字体大小
- 网格列数：2-20 列
- 每页数量：1-240 个卡片
- 顶部筛选：全部 / 视频 / 图片 / 收藏

## 本地与安全说明

- 工具以本地 Python 服务运行，地址是 `http://127.0.0.1:8787`。
- 工具不会把你的媒体文件上传到云端。
- 设置保存在 `config.json`，其中可能包含你的本地文件夹路径。
- 审核数据保存在本地 `review_data.json`。
- 移到回收站会先使用扫描目录内的项目回收文件夹 `_video_wall_trash`；可在回收站页面恢复原文件，或再移到 Windows 系统回收站。系统回收站未被禁用或清空时，通常也可以从 Windows 恢复。
- 500MB 以上视频默认不会在墙内直接预览，避免误加载大型电影或演唱会文件。
- 默认不会生成持久化图片缩略图缓存或视频封面缓存。

浏览器自动播放视频通常要求视频静音，所以瀑布流里的视频默认是静音播放。500MB 以上的视频默认在墙内显示占位封面，避免误加载电影或演唱会等超大文件；可在设置中修改。

视频文件的内嵌生成元数据取决于视频生成工具是否真的把信息写进容器。如果系统 PATH 中可以找到 `ffprobe`，工具会尝试读取 MP4/WebM/MOV 容器 tags，例如 prompt、workflow、comment、description、时长、编码和分辨率。如果没有安装 `ffprobe`，仍会安全回退到同名 sidecar JSON 和基础文件信息。

## 配置文件

工具会在本地使用：

```text
config.json
```

保存你的设置，例如路径、列数、每页数量、墙内播放上限、悬浮分页条、排序方式、沉浸模式、语言、主题和字体大小。

幻灯片的间隔、效果、Fit 模式和循环设置也会保存在 `config.json`。

这个文件可能包含你的本地路径，不建议上传到 GitHub。上传仓库时保留 `config.example.json` 即可。
